# Phishbowl

**An offline social-engineering simulator for red teamers.**

You're the attacker in a sealed sandbox — the "bowl." Given a briefing and an objective, you work a cast of AI-driven personas across three simulated apps (a messenger, a social feed, and an email client), using pretexting, phishing, and manipulation to reach your goal. Every character runs on a local LLM via [Ollama](https://ollama.com) with their own personality, knowledge, secrets, and a suspicion meter that climbs when you slip.

Unlike defensive phishing platforms (Gophish, KnowBe4, Arsen) that test *your* employees, Phishbowl puts you on the **offensive** side to sharpen the human-attack skill itself. Recon, pretext design, rapport-building, and pivoting.

---

## Ethics & Intended Use

Phishbowl is a **training tool**. It exists to help security professionals understand and practice social-engineering tradecraft in a fully simulated environment so they can better defend against it.

- **Everyone in Phishbowl is fictional.** All personas, companies, and data are generated by a local model. There are no real targets, accounts, or systems involved.
- **It runs entirely offline.** Phishbowl talks only to a local Ollama instance. It does not send messages, emails, or requests to any real person, service, or network.
- **It does not generate operational attack infrastructure.** No real phishing pages, no payloads, no credential-harvesting against live systems.
- **Intended users:** red teamers, penetration testers, security awareness trainers, students, and CTF builders.
- **Not intended for** conducting, planning, or rehearsing attacks against real people or organizations without explicit authorization.

By using Phishbowl you agree to use it for lawful, authorized, educational purposes only. The authors accept no liability for misuse.

## How it works

1. **Briefing.** You receive a scenario: the target org, your objective (e.g. "obtain VPN credentials from someone in Finance"), and any starting intel.
2. **Recon.** Explore the simulated social feed and directories to learn who's who, build a target list, and find pretext hooks.
3. **Engage.** Open conversations in the messenger or email client. Each NPC is a stateful LLM persona. They they remember what you've said, react to inconsistencies, and escalate suspicion.
4. **Pivot & escalate.** Use information from one persona to gain trust with another. Chain small wins into your objective.
5. **Outcome & scoring.** Win by achieving the objective. Lose if suspicion trips a "security team" response. Get scored on stealth, efficiency, and technique.

## Stack (provisional)

- **Shell:** Tauri (Rust) — small, native, offline desktop bundle
- **Frontend:** web UI rendering the three simulated apps
- **Backend:** Rust — NPC orchestration, world state, scoring, persistence (SQLite)
- **AI:** local LLMs via Ollama

*The stack may shift as design does*
